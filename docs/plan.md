# WCMS 개발 계획

> **현재 버전**: v0.8.6 (Released)
> **목표 버전**: v0.8.7

---

## [v0.8.7] - 클라이언트 자동 업데이트 및 프로그램 관리 강화

### 1. 클라이언트 자동 업데이트
- [ ] **자동 업데이트 로직 구현**
  - `client/main.py`: `check_for_updates` 함수 개선
  - 새 버전 감지 시 `install.cmd` 또는 별도의 업데이트 스크립트 다운로드 및 실행
  - 서비스 중지 -> 파일 교체 -> 서비스 시작 프로세스 자동화
- [ ] **업데이트 정책 설정**
  - 강제 업데이트 vs 선택적 업데이트
  - 업데이트 시간대 설정 (수업 시간 피하기)

### 2. 프로그램 삭제 기능 강화
- [ ] **Chocolatey 삭제**: `choco uninstall` 명령 지원
- [ ] **일반 프로그램(EXE/MSI) 삭제**:
  - PowerShell을 이용하여 레지스트리(`Uninstall` 키) 검색 및 언인스톨러 실행
  - 게임 등 Chocolatey로 설치되지 않은 프로그램 삭제 지원
- [ ] **서버 UI**: 프로그램 삭제 모달에서 삭제 방식(Chocolatey/일반) 선택 가능하게 개선

### 3. 프로그램 차단(Blacklist) 기능
- [ ] **차단 목록 관리**: 서버 관리자 페이지에서 차단할 프로세스 이름(예: `game.exe`) 등록
- [ ] **클라이언트 감시**:
  - 클라이언트가 주기적으로(또는 하트비트 시) 차단 목록 동기화
  - 실행 중인 프로세스 모니터링 및 차단된 프로세스 발견 시 즉시 강제 종료
  - 사용자에게 "차단된 프로그램입니다" 메시지 표시

## [Backlog] - 향후 계획

### 보안 강화 (DevSecOps)
- [ ] **취약점 검사 자동화**: OWASP ZAP을 활용한 DAST(동적 애플리케이션 보안 테스트) GitHub Actions 워크플로우 구축
- [ ] **코드 보안 분석**: CodeQL을 이용한 SAST(정적 애플리케이션 보안 테스트) 적용
- [ ] **HTTPS 지원**: Let's Encrypt 인증서 자동 발급 및 갱신
- [ ] **관리자 2FA**: OTP 기반 2단계 인증 도입
- [ ] **의존성 보안**: Dependabot 설정으로 라이브러리 취약점 관리

### 테스트 및 품질

- [ ] **CPU 부하 테스트**: 클라이언트에 CPU 부하를 발생시키는 명령 추가 (성능/안정성 테스트용)

### 기능 확장

- [ ] **파일 배포**: 서버에서 클라이언트로 파일 전송 기능 강화
