## 🐛 해결된 이슈

| 제목 | 해결 |
|---|---|
| DB 스키마 불일치 오류 | `pc_info` 테이블에 `mac_address` 컬럼 추가 |
| **불필요한 데이터 중복 전송** | **등록(정적)/하트비트(동적) 정보 전송 분리** |
| WMI 스레드 오류 | CoInitialize 추가 (경고는 여전히 발생하나 기능 정상) |
| disk_info 직렬화 오류 | JSON 변환 |
| IP/MAC 주소 고정값 | 실제 값 조회 |
| **좌석번호 null 문제** | **좌석 배치 저장시 pc_info.seat_number 자동 업데이트 추가** |

## 📝 주요 결정사항

1. **정적/동적 정보 분리**: 클라이언트의 부하를 줄이기 위해 등록 시에만 정적 정보를, 하트비트 시에는 동적 정보만 전송.
2. **프로세스 로깅**: 사용자가 직접 설치한 프로그램 위주로 필터링하여 `heartbeat`에 포함.
3. **Long-polling**: WebSocket 대신 간단한 30초 폴링으로 명령 시스템 구현.
4. **배포**: PyInstaller + GitHub Actions로 빌드 자동화.
5. **DB**: SQLite (경량, 로컬 테스트 용이).
